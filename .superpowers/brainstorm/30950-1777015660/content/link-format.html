PK链接如何传递数据？

用户A分享后，用户B打开链接时如何知道这是谁的PK挑战？

A

用户ID参数

链接格式：/pk?from=userId_A

B打开链接时，系统用A的用户ID查出A的报告数据

  • ✓ 实现最简单
  • ✓ 可复用现有用户报告
  • ✗ 链接暴露用户ID（无隐私风险，只是匿名标识）
B

PK Token

创建PK挑战时服务端生成唯一token，链接格式：/pk?challenge=token

token关联挑战者和挑战数据，对手填完生日后关联到同一token

  • ✓ 更规范，数据模型清晰
  • ✓ 可支持多人围观
  • ✗ 需要新建PK表
C

简短分享码

6位数字/字母分享码，链接格式：/pk/codered

用户A生成后保存到数据库，对手输入码或扫码参与

  • ✓ 链接更短更美观
  • ✓ 适合海报展示
  • ✗ 需要多一步"输入码"

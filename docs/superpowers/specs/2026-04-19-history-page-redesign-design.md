# 历史报告列表页大改版 + 账号体系设计

> **Goal:** 建立简单账号体系 + 重构历史报告页面，实现个人档案管理和报告对比功能

## 背景

当前历史页面 (`/history`) 存在以下问题：
1. 只支持 anonymous 用户，现有报告无法跨设备同步
2. 雷达图只显示 3 维（career/love/wealth），缺少 health/mentor
3. 无解锁状态标识
4. 无删除报告功能
5. 无报告对比能力

## 用户场景

个人档案管理：用户注册后，可管理自己及家人的多篇命盘报告，进行历史对比分析。

## 账号体系设计

### 数据模型

**新增 users 表：**
```sql
CREATE TABLE users (
  id TEXT PRIMARY KEY,
  phone TEXT UNIQUE NOT NULL,
  createdAt TEXT NOT NULL,
  lastLoginAt TEXT NOT NULL
);
```

**修改 reports 表 userId：**
- 原：`userId TEXT NOT NULL`（值为 "anonymous"）
- 新：`userId TEXT NOT NULL`（关联 users.id）

**迁移策略：**
- 现有 anonymous 报告通过设备 UUID 标识
- 用户登录后，可认领（claim）设备上的 anonymous 报告

### 登录流程（模拟）

1. 用户输入手机号 → 发送验证码
2. 前端生成 6 位模拟验证码（实际不发送短信）
3. 用户输入验证码 → 前端验证通过
4. 服务端创建/查找用户，返回 userId
5. userId 存储于 httpOnly cookie 或 localStorage

> **注意：** 短信验证为模拟实现。真实上线时需接入腾讯云/阿里云 SMS。

## 页面设计

### 登录页 `/login`

**交互流程：**
1. 用户输入手机号（11位中国大陆手机号）
2. 点击"获取验证码" → 60 秒倒计时
3. 输入 6 位验证码
4. 点击登录 → 验证并跳转


## API 设计

### POST /api/auth/login
**请求：**
```json
{ "phone": "13800138000", "code": "123456" }
```
**响应（成功）：**
```json
{ "userId": "usr_xxx", "phone": "138***8000" }
```
**响应（失败）：**
```json
{ "error": "验证码错误" }
```

### POST /api/auth/send-code
**请求：**
```json
{ "phone": "13800138000" }
```
**响应：**
```json
{ "success": true }
```
> 模拟版：前端直接返回 success，验证码固定为 `123456`

### GET /api/reports
**修改：** 优先从 cookie 获取 userId，fallback 到 anonymous

### DELETE /api/reports/:id
**功能：** 删除指定报告

## 数据库变更

### 迁移 SQL

```sql
-- 1. 创建用户表
CREATE TABLE IF NOT EXISTS users (
  id TEXT PRIMARY KEY,
  phone TEXT UNIQUE NOT NULL,
  createdAt TEXT NOT NULL,
  lastLoginAt TEXT NOT NULL
);

-- 2. reports 表已有 userId 字段，无需修改结构
-- 3. anonymous 报告保持 userId='anonymous'，登录后可认领
```

## 技术实现顺序

1. **登录模块** — `LoginForm` + `/api/auth/login` + `/api/auth/send-code`
2. **用户状态管理** — Context 或 Zustand 存储登录状态
3. **历史页面重构** — `HistoryList` + `ReportCard` + `MiniRadar`
4. **报告对比** — `CompareView` + 雷达图并列展示
5. **档案页** — `ProfilePage` + 家庭成员管理

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| 验证码发送失败 | Toast 提示"发送失败，请重试" |
| 验证码错误 | 输入框下方红色提示"验证码错误" |
| 登录超时/网络错误 | Toast 提示"网络异常，请重试" |
| 删除报告失败 | Toast 提示，报告保留在列表 |
| 报告不存在 | 跳转 404 页面 |

---

## 设计决策

1. **不引入复杂状态管理库** — 用 React Context 足够
2. **设备匿名报告保留** — 用户未登录时仍可生成报告，不强制登录
3. **验证码固定为 `123456`** — 模拟阶段简化实现
4. **家庭成员在客户端用 localStorage 存储** — 无需单独表，简单轻量
5. **对比只支持 2 篇报告同时对比** — 避免多篇对比界面过于复杂

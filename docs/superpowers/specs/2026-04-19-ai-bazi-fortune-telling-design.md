# AI 八字命理分析仪 — 设计文档

## 1. 产品定位

AI 八字命理分析仪是一款基于传统八字命理 + AI 的智能运势解读产品。用户输入出生信息，AI 追问补充后生成完整命盘报告，并以未来科技感的方式呈现。

**核心差异化：** 专业命理算法 + AI 深度解读 + 未来感视觉体验

## 3. 视觉风格

**主题：未来科技感 — 星象宇宙**

- 深色背景（近黑色 + 深蓝渐变）
- 星点粒子动画背景
- 流光/极光动效
- 主色调：深蓝 #0a0e27 + 星光金 #f0c674 + 流光紫 #7b68ee
- 字体：现代无衬线，中文用思源黑体或阿里巴巴普惠体
- 整体氛围：像在星空中解读生命的奥秘

## 4. 核心交互流程

```
用户进入 → 填写基本信息
  ↓
出生信息表单（姓名、性别、出生年月日时、出生地）
  ↓
AI 追问（"你想重点看事业还是感情？""你有特定想了解的问题吗？"）
  ↓
八字排盘算法计算 → 生成命盘数据
  ↓
AI 解读 → 生成完整报告
  ↓
展示：雷达图（各维度分数）+ 对话式解读
  ↓
历史记录保存（用户可回看）
```

## 5. 功能模块

### 5.1 用户输入

**表单字段：**
- 姓名（用于称呼）
- 性别
- 出生公历日期（年/月/日）
- 出生时间（时/分，24小时制）
- 出生地点（省市，用于真太阳时校正）

**AI 追问机制：**
- 表单提交后，AI 主动追问关注重点
- 最多 2-3 轮追问后开始生成报告
- 追问内容示例："你最想了解哪方面？事业 / 感情 / 财运 / 健康？""有什么具体困扰吗？"

### 5.2 八字排盘算法（自建）

核心计算模块，不依赖第三方付费 API：

1. **公历转农历** — 查万年历数据确定农历日期
2. **节气判断** — 精确到分钟，判断出生时刻属于哪个节气
3. **排八字** — 年柱、月柱、日柱、时柱的天干地支
4. **算大运** — 根据月令节气和性别起运，计算起运年龄
5. **流年计算** — 从起运年龄开始，每年对应一个地支
6. **日主旺衰** — 五行金木水火土的强弱分析
7. **十神推导** — 根据天干地支关系计算十神
8. **纳音五行** — 年柱纳音、月柱纳音等

> 排盘算法为确定性数学计算，非 AI 推理，每次结果一致。

### 5.3 AI 解读

**接入：** Minimax API（国内合规）

**解读维度：**
- 命局总评（性格、天赋、优劣势）
- 事业运（事业心、工作方向、贵人运）
- 感情运（桃花、伴侣缘分、感情挑战）
- 财运（赚钱能力、理财风格、破财风险）
- 健康运（重点关注的身体部位）
- 大运分析（未来10年趋势）
- 流年预测（当年运势详解）

**风格：**
- 自信直接，像老练命理师
- 负面预测带正向引导，不冷酷
- 每个维度有分数（雷达图）+ 文字解读

### 5.4 报告展示

**雷达图：**
- 5个维度：事业、感情、财运、健康、贵人
- 分数 0-100，可视化圆形雷达图
- 流光动画效果

**对话式解读：**
- AI 文字分段输出，每段一个维度
- 像命理师在旁边讲解
- 用户可展开/收起各维度详情

**大运/流年切换：**
- 时间轴视图，横向展示人生各阶段
- 点击可看该阶段详细解读

### 5.5 历史记录 & 保存

**数据库模型：**
```
User {
  id, openid/unionid, created_at
}

Report {
  id, user_id,
  name（姓名）, gender, birth_data（出生信息JSON）,
  bazi_data（八字排盘结果JSON）,
  ai_analysis（AI解读结果JSON）,
  radar_scores（雷达图分数JSON）,
  is_full_version（是否完整版）,
  unlocked（是否已解锁完整版）,
  created_at
}
```

**接口：**
- `POST /api/reports` — 保存报告
- `GET /api/reports?user_id=xxx` — 获取用户历史报告列表
- `GET /api/reports/:id` — 获取单条报告详情
- `POST /api/reports/:id/unlock` — 解锁完整版（付费）

## 6. 商业模式

**首次体验：** 免费生成完整报告（含所有维度 + 大运 + 流年）

**付费解锁点（后续）：**
- 单项深度分析（"我这年什么时候有桃花运"）
- 年度流年报告（每年更新）
- 高级命盘（叠加紫微斗数等）
- 命盘对比（合盘分析）

## 7. 技术架构

**前端：** Next.js + React + TailwindCSS

**后端：** Next.js API Routes

**数据库：** PostgreSQL（可先用 SQLite 早期验证）

**AI：** Minimax API（国内合规）

**八字算法：** 自建 Node.js 模块，基于开源万年历数据

**部署：** Vercel（Web一期）

## 8. 非功能要求

- 页面加载 < 3秒（首屏）
- 报告生成 < 10秒
- 雷达图动画流畅（60fps）
- 深色主题护眼
- 移动端适配（优先手机浏览）

---

## 9. 风险 & 挑战

| 风险 | 对策 |
|------|------|
| AI 幻觉导致命理解读不准确 | 提示用户仅供参考，AI 解读是娱乐性质 |
| 用户对"准不准"争议大 | 重点放在体验和视觉，解读风格要温暖正向 |
| 八字算法精度问题 | 用成熟开源算法 + 边界 case 测试 |
| AI API 成本上涨 | 早期用免费额度，量大后谈代理价 |

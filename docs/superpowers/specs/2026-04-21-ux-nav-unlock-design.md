# 移动端导航 & 报告解锁流程设计方案

> **Spec Version:** 1.0  
> **Date:** 2026-04-21  
> **Author:** UX Design Session

## 一、移动端底部导航栏

### 1.1 设计决策

| 决策项 | 选择 |
|--------|------|
| 标签数量 | 3个（首页、历史、我的） |
| 可见页面 | 除登录页外的所有页面 |
| 图标风格 | 金色渐变（与品牌色呼应） |
| 激活态 | 底部金色滑动指示条 |
| 背景 | 毛玻璃效果 |

### 1.2 视觉规范

**结构：**
```
┌─────────────────────────────────┐
│         页面内容                 │
│         (padding-bottom: 80px)   │
├─────────────────────────────────┤
│  [首页]    [历史]    [我的]      │  ← 毛玻璃背景 + 固定底部
│   ●        ○         ○          │  ← 激活态金色指示条
└─────────────────────────────────┘
```

**规格：**
- 高度：64px（含 safe-area）
- 图标尺寸：28×28px
- 标签文字：10px，激活态金色，未激活灰色
- 背景：`rgba(10, 10, 15, 0.85)` + `backdrop-filter: blur(12px)`
- 上边框：1px `rgba(255,255,255,0.06)`
- 指示条：高度2px，金色渐变，圆角1px，宽度等于标签文字宽度+图标宽度
- 指示条动画：transform 300ms cubic-bezier(0.4, 0, 0.2, 1)

**激活态：**
- 图标：金色渐变填充
- 文字：金色 (#d4af37)
- 指示条：显示

**未激活态：**
- 图标：灰色 (#666) 线框描边
- 文字：灰色 (#666)

### 1.3 组件结构

**新增组件：** `src/components/ui/BottomNav.tsx`

```typescript
interface NavItem {
  path: string;
  label: string;
  icon: 'home' | 'history' | 'profile';
}

const NAV_ITEMS: NavItem[] = [
  { path: '/', label: '首页', icon: 'home' },
  { path: '/history', label: '历史', icon: 'history' },
  { path: '/profile', label: '我的', icon: 'profile' },
];
```

**挂载位置：** `src/app/layout.tsx` — 放在 `PageTransition` 内部，`children` 下方

**可见性逻辑：** 通过 `usePathname()` 获取当前路径，登录页 (`/login`) 隐藏

### 1.4 路由映射

| 路径 | 底部导航激活 |
|------|-------------|
| `/` | 首页 |
| `/history` | 历史 |
| `/profile` | 我的 |
| `/report/[id]` | 继承父级（首页） |
| `/login` | 不显示 |

## 二、报告解锁流程

### 2.1 设计决策

| 决策项 | 选择 |
|--------|------|
| 展示位置 | 详情页内嵌底部面板 |
| 内容展示 | 增强版（清单 + 每项1-2句示例） |
| 交互 | 展开/收起 + 直接 CTA |
| 支付方式 | 模拟支付（直接解锁） |

### 2.2 解锁面板视觉规范

**结构：**
```
┌─────────────────────────────────┐
│  [锁定内容区 — 预览截断]          │
├─────────────────────────────────┤
│  ───── 解锁完整报告 ─────        │  ← 装饰分隔线
│                                 │
│  包含以下深度内容：              │
│  · 职业推荐 — 具体岗位方向         │  ← 示例内容
│  · 贵人方位 — 方位与时机          │
│  · 配偶特征 — 理想伴侣画像        │
│  · 婚恋建议 — 具体行动指南        │
│  · 幸运元素 — 颜色/数字/方位     │
│  · 起名建议 — 用字分析           │
│                                 │
│  ┌─────────────────────────┐   │
│  │   解锁完整报告 — ¥29      │   │  ← CTA 按钮
│  └─────────────────────────┘   │
│                                 │
│  [⋀ 收起]                       │  ← 收起按钮
└─────────────────────────────────┘
```

### 2.3 面板内容规范

**分隔装饰：**
- 渐变金色线，左右向中间汇聚，中间可有小菱形装饰
- 文字"解锁完整报告"居中，serif 字体

**内容清单：**
- 每项列出：分类名（加粗）+ 示例内容1-2句
- 示例内容用斜体或较淡颜色，与实际解锁内容一致
- 6项内容：职业推荐、贵人方位、配偶特征、婚恋建议、幸运元素（颜色/数字/方位）、起名建议

**CTA 按钮：**
- 渐变背景：`linear-gradient(135deg, var(--color-primary), var(--color-accent))`
- 文字："解锁完整报告 — ¥29"
- 高度：48px，全角宽度
- 圆角：24px（胶囊形）
- Hover：微上移 + 阴影加深
- Active：scale(0.98)
- Loading 态："处理中..." + 旋转 spinner

**收起功能：**
- 面板默认展开
- 底部有收起按钮，点击折叠面板
- 收起时只显示一行文字 + 展开箭头："解锁完整报告查看更多详情 ▼"
- 点击展开箭头可重新展开

### 2.4 解锁流程交互

**步骤：**
1. 用户点击"解锁完整报告 — ¥29"
2. 按钮变为 Loading 态："处理中..."
3. 调用 `POST /api/reports/[id]/unlock`
4. 成功后：按钮显示"已解锁 ✓"，短暂停留（800ms）后页面自动刷新
5. 刷新后：完整内容展示，解锁面板消失

**（可选增强）悲观 UI：**
- 调用成功后立即更新本地状态（isLocked=false），无需等待刷新
- 页面刷新是为了同步服务端数据

### 2.5 组件修改

**修改文件：**
- `src/components/FortuneDisplay.tsx` — 将现有的锁定消息面板替换为新的解锁面板组件
- `src/components/UnlockPanel.tsx` — 新建解锁面板组件

**解锁面板 Props：**
```typescript
interface UnlockPanelProps {
  reportId: string;
  hiddenSections: string[]; // e.g. ['career', 'mentor', 'love', ...]
  isLocked: boolean;
}
```

## 三、通用规范

### 3.1 动画时间

| 场景 | 时长 | 缓动 |
|------|------|------|
| 指示条滑动 | 300ms | cubic-bezier(0.4, 0, 0.2, 1) |
| 按钮 hover | 150ms | ease-out |
| 面板展开/收起 | 300ms | var(--ease-smooth) |
| 解锁成功反馈 | 800ms | ease-out |

### 3.2 色彩变量

```css
--color-primary: #8b5cf6;       /* 主色（紫色） */
--color-accent: #c41e3a;        /* 点缀色（深红） */
--color-gold: #d4af37;          /* 金色 */
--color-gold-light: #f0c674;    /* 浅金色 */
--color-bg-page: #0a0a0f;       /* 页面背景 */
--color-surface: rgba(20,20,28,0.85); /* 毛玻璃背景 */
```

### 3.3 安全区域

底部导航需要处理 iOS 刘海屏：
```css
padding-bottom: env(safe-area-inset-bottom);
```

# 出生信息表单优化设计方案

**日期：** 2026-04-22
**类型：** UI/UX 优化
**优先级：** P0

---

## 1. 背景

当前出生信息表单存在以下问题：
- 字段过多（7个），用户认知负担重
- 年/月/日三个独立输入框，体验割裂
- 分钟精确到个位数，但大多数用户不记得出生时的精确分钟

**目标：** 减少字段数量至5个，降低用户认知负担，同时不影响八字计算的准确性。

---

## 2. 改动范围

### 2.1 出生日期 → 原生 date picker

**现状：**
- 年、月、日三个独立 `<input type="number">`
- 用户需手动输入或点击微调

**优化后：**
- 合并为单个 `<input type="date">`
- 移动端触发系统原生 date picker
- 年份范围：1900-01-01 至 今天

**代码改动：**
```tsx
// 替换三个独立输入框
<input
  type="date"
  value={form.birthDate} // YYYY-MM-DD 格式
  onChange={e => {
    const date = new Date(e.target.value);
    setForm({
      ...form,
      year: date.getFullYear(),
      month: date.getMonth() + 1,
      day: date.getDate(),
    });
  }}
  className="w-full rounded-[var(--radius-md)] px-5 py-4 text-white ..."
  style={{
    colorScheme: 'dark',
    background: 'var(--color-surface)',
    borderColor: errors.date ? 'var(--color-error)' : 'var(--color-border)',
  }}
/>
```

**CSS 适配：**
```css
input[type="date"]::-webkit-calendar-picker-indicator {
  filter: invert(0.8);
}
input[type="date"] {
  color-scheme: dark;
}
```

### 2.2 出生时辰 → 保留地支选择 + 早/中/晚三段

**现状：**
- 时辰下拉框只显示地支名称和完整时间范围

**优化后：**
- 时辰下拉框保持不变
- 选中时辰后，显示"早/中/晚"三段选择按钮

**三段定义：**
| 时辰 | 早 | 中 | 晚 |
|------|-----|-----|-----|
| 子时(23-01) | 23:00-23:59 | 00:00-00:29 | 00:30-00:59 |
| 丑时(01-03) | 01:00-01:59 | 02:00-02:29 | 02:30-02:59 |
| 寅时(03-05) | 03:00-03:59 | 04:00-04:29 | 04:30-04:59 |
| 卯时(05-07) | 05:00-05:59 | 06:00-06:29 | 06:30-06:59 |
| 辰时(07-09) | 07:00-07:59 | 08:00-08:29 | 08:30-08:59 |
| 巳时(09-11) | 09:00-09:59 | 10:00-10:29 | 10:30-10:59 |
| 午时(11-13) | 11:00-11:59 | 12:00-12:29 | 12:30-12:59 |
| 未时(13-15) | 13:00-13:59 | 14:00-14:29 | 14:30-14:59 |
| 申时(15-17) | 15:00-15:59 | 16:00-16:29 | 16:30-16:59 |
| 酉时(17-19) | 17:00-17:59 | 18:00-18:29 | 18:30-18:59 |
| 戌时(19-21) | 19:00-19:59 | 20:00-20:29 | 20:30-20:59 |
| 亥时(21-23) | 21:00-21:59 | 22:00-22:29 | 22:30-22:59 |

**数据结构变更：**
```typescript
interface BirthFormData {
  name: string;
  gender: 'male' | 'female';
  year: number;
  month: number;
  day: number;
  hour: number;      // 时钟小时（0-23），地支由 Math.floor(hour/2)%12 计算
  minute: number;    // 保留但固定为0，不在UI显示
  province: string;
  // 新增字段
  timeSegment: 'early' | 'middle' | 'late'; // 早/中/晚
}
```

**交互逻辑：**
1. 用户选择地支（如"丑时"）
2. 显示三段选择按钮，默认选中"中"
3. 根据选择的段，计算实际时钟小时
   - 丑时早 → hour: 1
   - 丑时中 → hour: 2
   - 丑时晚 → hour: 2（但 minute 不同）

**视觉设计：**
```
┌─────────────────────────────────────┐
│  出生时辰                            │
│  ┌─────────────────────────────────┐ │
│  │ 丑时（01:00-02:59）          ▼  │ │
│  └─────────────────────────────────┘ │
│                                      │
│  ┌───────┐ ┌───────┐ ┌───────┐     │
│  │  早   │ │  中   │ │  晚   │     │
│  └───────┘ └───────┘ └───────┘     │
└─────────────────────────────────────┘
```

### 2.3 分钟字段 → 从 UI 移除

**现状：**
- 独立分钟输入框（0-59）
- 精确到分钟

**优化后：**
- 从 UI 完全移除分钟输入
- 数据结构中仍保留 `minute: 0`（API 需要）

**注意：** 这是一个 breaking change 的规避方案。API 已经处理了默认值：
```typescript
// api/ai/analyze/route.ts
minute: birthData.minute || 0,  // 默认值为 0
```

## 3. 最终表单结构

```
出生信息
├── 姓名        [输入框]
├── 性别        [男/女 切换按钮]
├── 出生日期    [原生 date picker]
├── 出生时辰    [地支下拉 + 早/中/晚按钮]
└── 出生省份    [省份下拉]
    [提交按钮]
```

**字段变化：7 → 5 个**

---

## 4. 不受影响的功能

### 4.1 API 和八字计算
- `minute` 字段在 API 层已有默认值 0，不影响计算
- `hour` 字段逻辑保持不变
- 八字库 `calculateBaZi()` 函数无需修改

### 4.2 数据库
- `BirthFormData` 接口添加 `timeSegment` 字段
- 历史数据可正常读取（`timeSegment` 为 undefined 时默认"中"）

---

## 5. 待后续处理的关联问题

### 5.1 hour 值概念不一致（潜在 bug）
当前表单的 `hour` 值（1, 3, 5...）与八字库期望的时钟小时（1, 3, 5...）恰好一致，但这是因为：
- 丑时起点是 1:00 → hour = 1
- 寅时起点是 3:00 → hour = 3
- ...

如果用户选择"丑时早"（1:00-1:59），hour 值应该是 1
如果用户选择"丑时中"（2:00-2:29），hour 值应该是 2

**建议后续单独提一个 bugfix 修复此问题。**
